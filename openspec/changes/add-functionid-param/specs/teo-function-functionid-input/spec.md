## ADDED Requirements

### Requirement: User can specify FunctionId when creating tencentcloud_teo_function resource

The system SHALL allow users to specify a custom FunctionId when creating a tencentcloud_teo_function resource. When FunctionId is not specified, the system SHALL use the FunctionId generated by the CreateFunction API. The function_id field SHALL be defined as both Optional and Computed in the resource schema.

#### Scenario: Create resource with custom FunctionId
- **WHEN** user creates a tencentcloud_teo_function resource with a custom function_id value
- **THEN** system SHALL pass the specified FunctionId to CreateFunction API
- **THEN** system SHALL successfully create the resource with the specified FunctionId
- **THEN** system SHALL set the resource ID to "zoneId#functionId"
- **THEN** resource read SHALL return the user-specified FunctionId

#### Scenario: Create resource without specifying FunctionId (backward compatibility)
- **WHEN** user creates a tencentcloud_teo_function resource without specifying function_id
- **THEN** system SHALL NOT pass FunctionId parameter to CreateFunction API
- **THEN** CreateFunction API SHALL generate a FunctionId
- **THEN** system SHALL use the API-generated FunctionId
- **THEN** resource ID SHALL be set to "zoneId#generatedFunctionId"
- **THEN** resource read SHALL return the API-generated FunctionId

#### Scenario: Update resource with FunctionId already set
- **WHEN** user updates a tencentcloud_teo_function resource that was created with a custom FunctionId
- **THEN** system SHALL maintain the same FunctionId
- **THEN** function_id SHALL remain read-only during update (immutable)
- **THEN** update SHALL only modify mutable fields like remark and content

#### Scenario: Import resource with existing FunctionId
- **WHEN** user imports an existing tencentcloud_teo_function resource
- **THEN** system SHALL read the existing FunctionId from the cloud
- **THEN** system SHALL set the resource ID to "zoneId#existingFunctionId"
- **THEN** resource state SHALL match the cloud state

#### Scenario: Validation of FunctionId format
- **WHEN** user provides a FunctionId with invalid format
- **THEN** CreateFunction API SHALL return a validation error
- **THEN** system SHALL propagate the error to the user
- **THEN** resource creation SHALL fail with clear error message

#### Scenario: Duplicate FunctionId
- **WHEN** user provides a FunctionId that already exists in the same zone
- **THEN** CreateFunction API SHALL return a conflict error
- **THEN** system SHALL propagate the error to the user
- **THEN** resource creation SHALL fail with clear error message
