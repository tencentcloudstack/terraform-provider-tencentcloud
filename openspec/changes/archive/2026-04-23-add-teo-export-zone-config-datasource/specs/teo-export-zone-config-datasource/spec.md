## ADDED Requirements

### Requirement: Export TEO zone configuration via data source
The system SHALL provide a Terraform data source `tencentcloud_teo_export_zone_config` that allows users to export TEO site configuration by calling the `ExportZoneConfig` API.

#### Scenario: Export zone configuration with zone_id only
- **WHEN** user provides valid `zone_id` without specifying `types`
- **THEN** system SHALL call the ExportZoneConfig API with only `zone_id`
- **THEN** system SHALL return all types of zone configuration in the `content` field

#### Scenario: Export zone configuration with specific types
- **WHEN** user provides valid `zone_id` and specifies `types` as `["L7AccelerationConfig"]`
- **THEN** system SHALL call the ExportZoneConfig API with both `zone_id` and `types`
- **THEN** system SHALL return only L7 acceleration configuration in the `content` field

#### Scenario: Export zone configuration with WebSecurity type
- **WHEN** user provides valid `zone_id` and specifies `types` as `["WebSecurity"]`
- **THEN** system SHALL call the ExportZoneConfig API with both `zone_id` and `types`
- **THEN** system SHALL return only Web security configuration in the `content` field

#### Scenario: Export zone configuration with multiple types
- **WHEN** user provides valid `zone_id` and specifies `types` as `["L7AccelerationConfig", "WebSecurity"]`
- **THEN** system SHALL call the ExportZoneConfig API with both `zone_id` and `types`
- **THEN** system SHALL return both L7 acceleration and Web security configuration in the `content` field

#### Scenario: Validate required zone_id parameter
- **WHEN** user omits the required `zone_id` parameter
- **THEN** system SHALL return a validation error indicating that `zone_id` is required

#### Scenario: Handle API errors with retry
- **WHEN** the ExportZoneConfig API returns a transient error
- **THEN** the service layer (`ExportZoneConfigByFilter`) SHALL retry the API call within `tccommon.ReadRetryTimeout`
- **THEN** the data source Read function SHALL directly return the error from the service layer without additional retry wrapping

#### Scenario: Return content as JSON string
- **WHEN** system receives a successful response from the ExportZoneConfig API
- **THEN** system SHALL set the `content` field to the value of `response.Response.Content`
- **THEN** the `content` field SHALL contain a JSON-encoded string of the zone configuration

#### Scenario: Set data source ID
- **WHEN** system successfully reads the zone configuration
- **THEN** system SHALL set the data source ID to a random token generated by `helper.BuildToken()`

#### Scenario: Save results to output file
- **WHEN** user specifies `result_output_file` parameter
- **THEN** system SHALL write the configuration results to the specified file path

#### Scenario: Register data source in provider
- **WHEN** the Terraform provider is initialized
- **THEN** system SHALL register `tencentcloud_teo_export_zone_config` as a data source in `provider.go`
- **THEN** system SHALL list `tencentcloud_teo_export_zone_config` in `provider.md`
