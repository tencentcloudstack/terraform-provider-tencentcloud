## Why

The `tencentcloud_elasticsearch_instance` resource currently cannot enable or disable the cluster destroy protection feature, which prevents accidental deletion of production Elasticsearch clusters. The Tencent Cloud ES `UpdateInstance` API already supports an `EnableDestroyProtection` input parameter (`OPEN`/`CLOSE`), and the `DescribeInstances` API returns the current `EnableDestroyProtection` value on `InstanceInfo`, but this field is not yet exposed in the Terraform resource schema. Users are forced to toggle destroy protection through the console or API outside of Terraform, breaking Infrastructure as Code coverage and risking unintended destruction.

## What Changes

- Add a new `enable_destroy_protection` parameter (string, `Optional` + `Computed`) to the `tencentcloud_elasticsearch_instance` resource schema. Valid values are `OPEN` (enable protection) and `CLOSE` (disable protection).
- In the resource `Update` method, detect changes to `enable_destroy_protection` and call the ES `UpdateInstance` API with the `EnableDestroyProtection` request field (with retry + upgrade-wait handling consistent with existing update logic).
- In the resource `Read` method, read the `EnableDestroyProtection` field from the `DescribeInstances` response (`InstanceInfo`) and set it into state (nil-safe).
- Extend the ES service-layer `UpdateInstance` wrapper signature in `service_tencentcloud_elasticsearch.go` to accept and forward an `enableDestroyProtection` string parameter to the `UpdateInstanceRequest.EnableDestroyProtection` field.
- Update the source documentation file `resource_tc_elasticsearch_instance.md` with an example showing the new parameter.
- Generate `website/docs/r/elasticsearch_instance.html.markdown` via `make doc`.
- Add a changelog entry under `.changelog/`.

## Capabilities

### New Capabilities
- `es-instance-destroy-protection`: Adds the ability to manage the destroy protection (`EnableDestroyProtection`) setting of a Tencent Cloud ES (Elasticsearch Service) instance through the `tencentcloud_elasticsearch_instance` Terraform resource, including update and read flows.

## Impact

- **Code**:
  - `tencentcloud/services/es/resource_tc_elasticsearch_instance.go` — add schema field, update `Read` and `Update` logic.
  - `tencentcloud/services/es/service_tencentcloud_elasticsearch.go` — extend `UpdateInstance` wrapper signature with `enableDestroyProtection` parameter; update all call sites.
  - `tencentcloud/services/es/resource_tc_elasticsearch_instance_test.go` — add unit tests (mock-based with gomonkey) for the new parameter.
  - `tencentcloud/services/es/resource_tc_elasticsearch_instance.md` — add usage example.
  - `website/docs/r/elasticsearch_instance.html.markdown` — generated by `make doc`.
  - `.changelog/<number>.txt` — changelog entry.
- **APIs**: Tencent Cloud ES `UpdateInstance` (input `EnableDestroyProtection`) and `DescribeInstances` (output `EnableDestroyProtection` on `InstanceInfo`). Both fields are already present in the vendored SDK (`github.com/tencentcloud/tencentcloud-sdk-go/tencentcloud/es/v20180416`), so no SDK upgrade is required.
- **Backward Compatibility**: The new parameter is `Optional` + `Computed`; existing configurations and state are unaffected. No breaking changes.
